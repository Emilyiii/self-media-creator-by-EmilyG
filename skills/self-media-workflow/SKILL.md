# Self-Media Workflow

## 适用场景
当用户要创作公众号、小红书、知识分享类内容，且需要与 Agent 协作推进，而不是一次性全自动产出时使用。

## 核心原则
- 不假设“一次成稿”
- 关键节点必须与用户协作
- 优先产出可讨论版本，再逐步收敛到终稿
- Skill 负责流程标准，Agent 负责执行与判断

## 标准流程

### 阶段 1：需求澄清
输入：主题、方向、平台、目标读者、语气偏好
输出：清晰创作任务定义

### 阶段 2：选题设计
调用选题类 Skill，给出 3 个候选方向。
要求：标题、切入角度、适配平台、建议写法。
用户在此节点选择或改题。

### 阶段 3：研究与提纲
调用 research Skill 收集材料，并输出提纲。
要求：
- 材料来源清晰
- 避免假数据
- 给出文章结构建议

### 阶段 4：初稿生成
调用写作 Skill 产出第一版正文。
要求：
- 先求可讨论，不求一步到位
- 明确文章核心论点
- 平台语气正确

### 阶段 5：终稿润色
调用润色 Skill：
- 去模板腔
- 去报告腔
- 拉近到用户真实表达风格
- 修顺节奏与过渡

### 阶段 6：配图规划（可选）
调用配图 Skill：
- 判断是否需要图
- 控制张数
- 给出图片位置和中英 prompt

### 阶段 7：排版与交付
调用排版 Skill 输出：
- Markdown 排版版
- HTML 版（如需要）
- 配图方案（如需要）

## 关键协作点
必须让用户参与的节点：
1. 选题确认
2. 初稿方向修正（如有必要）
3. 终稿确认

## 禁止事项
- 不要用 Python 脚本模拟 Agent 思考
- 不要通过中间任务文件绕路触发 LLM
- 不要把中间态垃圾文件当成长期资产

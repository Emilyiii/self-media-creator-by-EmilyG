# Topic Planning

## 适用场景
当用户只有大方向、模糊主题、灵感碎片，或者明确说“帮我想选题/角度/标题”时使用。

## 目标
不是“随机生成 3 个标题”，而是帮助用户：
- 明确写给谁
- 为什么值得写
- 从哪个角度切入更有传播性和完成度

## 标准输出
每次至少给出 3 个候选题，且每个题包含：
1. 标题
2. 切入角度
3. 适合平台（公众号 / 小红书 / 通用）
4. 建议写法（经验型 / 教程型 / 观点型 / 清单型）
5. 风险提示（是否太泛、太空、太容易写成模板文）

## 选题原则
- 优先具体，不要宏大空泛
- 优先有真实场景，不要纯概念
- 优先能展开成完整结构，不要只有一个标题感
- 优先让用户容易表达，不要逼用户写自己不熟的内容

## 协作要求
- 选题阶段必须允许用户改题、混搭、重出
- 用户未确认前，不进入正文写作
- 如果用户已经有明确题目，此 Skill 退化为“角度校准器”
